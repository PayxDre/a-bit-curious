Bitcoin mining sounds complicated, but the basic idea is simple: miners are computers that compete to solve a mathematical puzzleThey're trying to find a number (called a "nonce") that, when combined with the block's data and run through a SHA-256 hash function, produces a hash starting with a certain number of zeros. to earn the right to add the next block. The first one to solve it wins a reward in Bitcoin — the block reward, currently 3.125 BTC per block[25].

Here's the genius: mining makes adding transactions expensive. To cheat, you'd need to redo all that expensive work — and faster than the entire rest of the network. This is called Proof of WorkThe system where you prove you did hard mathematical work to earn the right to add a block. The work is expensive to do but cheap to verify..

Modern mining uses dedicated chips called ASICsApplication-Specific Integrated Circuit — a chip designed to do ONE thing: calculate Bitcoin hashes as fast as possible.. Many miners are now located near cheap, renewable energy sources. A new block is found roughly every 10 minutesBitcoin is designed to always target 10 minutes per block. If blocks come too fast, difficulty increases. Too slow, difficulty decreases. — by design.

⛓ The Blockchain

// The world's most transparent record book

The blockchain is Bitcoin's record book — but it's not like any you've ever seen. Imagine a notebook where every page is connected to the previous page with a mathematical lock. If you tried to change something on page 50, you'd break the lock connecting it to page 51, which breaks the lock to page 52, all the way to today's page. That's why it's called a blockchain — a chain of blocks of data, each locked to the one before it.

The blockchain is immutableImmutable means "cannot be changed." Once a transaction is written into the blockchain and several blocks are added on top, it is practically impossible to alter.. Every single Bitcoin transaction ever made — going back to the first one on January 12, 2009 — is permanently recorded and visible to anyone on a block explorerA website like mempool.space or blockstream.info that lets you browse the blockchain like a search engine..

Bitcoin's blockchain is completely public and transparent — but interestingly, pseudonymous. Addresses are visible to everyone, but they're not automatically linked to real-world identities. Every node has a complete copy. There's no central database that can be shut down.

The blockchain grows by one block roughly every 10 minutes, 24/7, forever. Each block contains hundreds or thousands of transactions. As of 2026, the blockchain is hundreds of gigabytes[31] and contains billions of transactions — all permanently, immutably, and transparently recorded.

🎯 Difficulty

// Bitcoin's self-balancing superpower

No matter how many miners join or leave, a new block is always found approximately every 10 minutes. Bitcoin has a built-in difficulty adjustmentEvery 2,016 blocks (~2 weeks), Bitcoin automatically measures how fast blocks have been coming in and adjusts the puzzle difficulty. that recalibrates every 2,016 blocks (~two weeks).

Think of it like a school exam that automatically adjusts. If too many students get answers too quickly, questions get harder. If students drop out, questions get easier. No one is in charge.

📤 Outputs

// How Bitcoin is actually "stored"

In Bitcoin, there's no such thing as a "balance." Instead, Bitcoin exists as UTXOsUnspent Transaction Outputs — the technical term for "Bitcoin you've received but haven't spent yet." Each UTXO is like a distinct physical coin. Your "balance" is just the sum of all your UTXOs. — Unspent Transaction Outputs. Think of it like a piggy bank full of physical coins and bills of different denominations.

When Alice sends 0.3 BTC to Bob, a new UTXO is created: "0.3 BTC locked to Bob's address." When Bob spends it later, he creates a new transaction that references this UTXO as input and creates new outputs. The old UTXO is "consumed."

⚡ SegWit

// Bitcoin's clever upgrade that made transactions cheaper

SegWitShort for "Segregated Witness" — activated August 2017. Moved signature data to a separate, discounted data area. stands for "Segregated Witness" — a fundamental upgrade activated in August 2017. Before SegWit, signature data (often 60%+ of transaction size) took up precious block space at full price. SegWit introduced witness data counting as only 0.25× its actual size toward the block limit — effectively increasing transactions per block by up to 4× without changing the base block size.
